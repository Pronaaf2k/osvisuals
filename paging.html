Virtual Memory: Paging and Swapping

Paging breaks memory into fixed-size pieces.

Instead of requiring contiguous physical memory, the OS maps virtual pages to any available physical frames. Swapping lets inactive pages live on disk until needed.

Core Vocabulary

Virtual pages become physical frames.

Page

Virtual chunk

Minimal unit of a process address space. With 12 offset bits, page size is 2^12 = 4 KiB.

Frame

Physical chunk

A physical page-sized slot in RAM that can hold any virtual page.

PTE

Mapping record

Stores frame number plus valid, present, dirty, referenced, and protection bits.

TLB

Translation cache

Avoids repeated page table walks by remembering recent translations.

Size Tension

Page size is a trade-off.

Large pages

Fewer PTEs and better TLB reach, but more internal fragmentation.

Small pages

Less wasted space per page, but more pages and more TLB misses.

Flat Page Table Cost

4 GiB / 4 KiB = 2^20 entries.

With 4-byte PTEs, one flat page table for a 32-bit process is 2^20 × 4B = 4 MiB. For 64-bit address spaces, flat tables become infeasible, so modern systems use multi-level tables.

32-bit VA: 10 bits level 1 | 10 bits level 2 | 12 bits offset
