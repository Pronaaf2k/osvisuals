CSE 323: Virtual Memory

Segmentation turns one process into movable memory regions.

The OS gives each process the illusion of a private address space. The MMU makes that illusion real by translating virtual addresses into physical addresses and blocking illegal accesses.

If processes share physical memory directly, one bad pointer can read or overwrite another process. Segmentation adds hardware checks between every program address and real memory.

Interactive Model

Base-only translation is fast, but unsafe.

A base register simply adds an offset: physical address = base + virtual address. Without a bounds check, a process can create a large virtual address and walk into another process.

Segmentation

Instead of one base and bounds pair, use one pair per region.

A process is not one solid block. Code, heap, and stack have different purposes, permissions, sizes, and growth directions. Segmentation maps each logical region independently.

Code segment: read and execute; usually does not grow.
Heap segment: read and write; grows upward as malloc needs space.
Stack segment: read and write; grows downward as calls allocate frames.
