Concurrency: Condition Variables and Semaphores

Locks protect data. CVs and semaphores coordinate timing.

A thread often needs to wait until something becomes true: a child exits, a buffer becomes non-empty, or a slot becomes free. These primitives let waiting threads sleep instead of spin.

See CV Wait Try Producer/Consumer

Primitive Cheat Sheet

Lock

Mutual exclusion for shared data.

CV

Queue for waiting on a condition.

Semaphore

CV-like wait queue plus integer state.

while (!condition)
  pthread_cond_wait(&cv, &mutex);

// wake one waiter
pthread_cond_signal(&cv);

Condition Variables

Wait atomically unlocks, sleeps, then re-locks.

The mutex prevents missed updates. The while loop prevents a woken thread from assuming the condition is still true.

Parent waits for child exit

Parent thread

checks done == false

Child thread

running work

Shared state

done = false

Why while, not if?

If is unsafe

After a signal, another thread may consume the condition before the woken thread runs.

While is safe

Every wakeup rechecks the condition under the mutex before continuing.

pthread_mutex_lock(&m);
while (!done)
  pthread_cond_wait(&c, &m);
pthread_mutex_unlock(&m);

Producer Consumer

Bounded buffers need both mutual exclusion and ordering.

Buffer Simulator

Buffer starts empty. Consumers wait on full/fill.

Correct CV Structure

Producer

while (count == MAX)
  wait(&empty, &mutex);
put(item);
signal(&fill);

Consumer

while (count == 0)
  wait(&fill, &mutex);
item = get();
signal(&empty);

Use separate condition variables for directed signaling: producers wait for empty slots, consumers wait for filled slots.

Semaphores

A semaphore is a counter with a wait queue.

sem_wait

Wait until value > 0, then decrement.

sem_post

Increment value and wake one waiter.

Producer-consumer semaphores

psem = BUFSIZE, csem = 0

Interchangeability

These primitives can implement each other.

Mutex from semaphore: initialize semaphore to 1, wait before critical section, post after.
Semaphore from CV + lock: protect an integer value and sleep while value <= 0.
Performance differs: choose based on contention, wait time, and critical-section frequency.

Many readers can share. Writers need exclusivity.
